## 🗺️ The business: Matir Doctor

"Matir Doctor" (মাটির ডাক্তার — "Soil Doctor") is FairFarm BD's IoT soil
sensor line for smallholder farmers, sold in three tiers:

| Device | Sensors | Price (BDT) |
|---|---|---|
| Matir Doctor Lite | Moisture, Temperature | 2,500 |
| Matir Doctor Pro | Moisture, pH, Temperature, Humidity | 4,800 |
| Matir Doctor Pro+ (NPK) | Moisture, pH, NPK, Temperature, Humidity | 7,900 |

Farmers across all 8 divisions of Bangladesh buy devices via the FairFarm
app/website, a field dealer network, or Facebook Shop; the devices stream
soil telemetry; and a support team handles setup, connectivity, and
calibration tickets.

## 🧱 Data model (star schema)

```
                         ┌───────────────┐
                         │   dim_date    │
                         └───────┬───────┘
                                 │
   ┌────────────┐      ┌────────┴────────┐      ┌──────────────┐
   │ dim_product │──────│   fact_sales    │──────│ dim_customer │
   └────────────┘      └────────┬────────┘      └──────┬───────┘
                                 │                       │
                         ┌───────┴────────┐      ┌───────┴────────┐
                         │  dim_division   │      │ fact_support_  │
                         │  / dim_district │      │    tickets     │
                         └───────┬────────┘      └────────────────┘
                                 │
                  ┌──────────────┴──────────────┐
                  │                              │
          ┌───────┴────────┐           ┌────────┴─────────┐
          │ fact_web_traffic│           │ fact_iot_readings │
          └────────────────┘           └───────────────────┘
```

**5 dimensions, 4 fact tables, ~84,000 rows**, generated by
`scripts/01_generate_synthetic_data.py` with realistic Bangladesh
geography (8 divisions, 29 districts), agricultural seasonality, and a
built-in **Dec-2025 "UX overhaul" breakpoint** so the before/after
analysis has genuine signal to detect.

| Table | Rows | Description |
|---|---|---|
| `dim_date` | 485 | Calendar table w/ agri-season & pre/post-UX flag |
| `dim_division` / `dim_district` | 8 / 29 | Bangladesh geography + lat/long |
| `dim_product` | 3 | Matir Doctor device tiers |
| `dim_customer` | 1,500 | Farmers: farm size, crop, segment, acquisition channel |
| `fact_sales` | 4,200 | Orders: channel, payment method, revenue |
| `fact_web_traffic` | 27,160 | Daily sessions/conversions by division & channel |
| `fact_iot_readings` | 49,583 | Weekly soil moisture/pH/NPK/battery telemetry |
| `fact_support_tickets` | 1,250 | Setup/connectivity/billing tickets, CSAT |

## 🔍 Key insights

| Metric | Pre-UX Overhaul | Post-UX Overhaul | Lift |
|---|---|---|---|
| Avg. conversion rate | 1.85% | 2.31% | **+24.9%** |
| Avg. daily sessions | 1,903 | 2,472 | **+29.9%** |

A Welch's t-test on daily conversion rates confirms the shift is
statistically significant (p < 0.001) — see
`docs/eda_kpi_summary.json` and `scripts/02_data_cleaning_eda.py`.

- **Total revenue:** BDT 29.7M across 4,200 orders
- **Top division by revenue:** Rajshahi (BDT 4.77M), followed by Dhaka and Khulna
- **Best-selling device:** Matir Doctor Pro (44% of revenue)
- **IoT fleet health:** 4.7% of sensor readings trigger an alert (low battery,
  pH out of range, or moisture stress) — mostly concentrated in Chattogram & Mymensingh
- **Support SLA:** average resolution time dropped after the reporting-process
  fix (see `Avg Resolution Time (hrs, Pre-Fix)` vs `(Post-Fix)` DAX measures)

## ▶️ Reproduce it yourself

```bash
pip install -r requirements.txt

# 1. Generate the synthetic star-schema dataset + messy raw export
python scripts/01_generate_synthetic_data.py

# 2. Clean the messy export, run EDA, validate the before/after KPI story
python scripts/02_data_cleaning_eda.py

# 3. Build the SQLite database for the SQL layer
python scripts/build_sqlite_db.py

# 4. Build the live Excel dashboard
python scripts/build_excel_dashboard.py

# 5. (optional) re-render the dashboard preview image — requires Playwright
python scripts/render_dashboard_preview.py
```

Run the SQL queries directly against the prebuilt database:
```bash
sqlite3 data/fairfarm_bd.db < sql/fairfarm_analysis_queries.sql
```

## 📈 Power BI dashboard

A `.pbix` is a Windows-only proprietary binary that can only be authored in
Power BI Desktop, so this repo ships everything needed to **build the real
file in under an hour** rather than a placeholder: paste-in Power Query M
(`powerbi/PowerQuery_M_Scripts.txt`), 25+ DAX measures
(`powerbi/DAX_Measures.txt`), a custom importable color theme
(`powerbi/FairFarm_BD_Theme.json`), and a full page-by-page build guide
(`powerbi/PowerBI_Build_Guide.md`). The preview image at the top of this
README shows the target design.
